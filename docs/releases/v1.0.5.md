# AILIS v1.0.5 Release Notes

Release date: 2026-06-22

## Highlights

- Renamed the desktop assistant/tooling surface toward AILIS across runtime code, docs, tests, and UI copy.
- Added Codex-style deferred tool discovery so the model starts with a small core tool surface and exposes MCP/Web/research tools on demand.
- Hardened tool schema contracts for required parameters, direct MCP calls, and invalid/empty tool arguments.
- Improved Web/Search evidence handling with evidence-gap reporting, LLM evidence audit requirements, Crawl4AI-style rendered fetch fallback, and local bundled web runtime preparation.
- Added GAIA-derived repair and spend-safety loops to reduce runaway iterations and make failed task chains easier to analyze.
- Added memory quality improvements, including larger relevant-memory context and control-panel memory clearing.
- Added local model provider support paths for OpenAI-compatible APIs, vLLM, Ollama-oriented deployment, and one-click local model setup guidance.
- Improved voice/TTS defaults so release builds do not unexpectedly speak malformed action/emote text; CosyVoice3 now uses an explicit local runtime bootstrap path.
- Added AILIS app icons, tray assets, and favicon resources for desktop and web presentation.

## Known Boundaries

- Vision screenshot understanding still requires a vision-capable provider/model. Text-only providers can capture screenshots but cannot run `describe_image`.
- Web/Search quality is much better structured than earlier builds, but high-confidence answers should still depend on fetched/parsed evidence instead of search-result snippets alone.
- GAIA auto-optimization is intentionally safety-gated; it should not be run as an unlimited API-spend loop.
- The v1.0.5 Windows release publishes the portable build. A full NSIS installer should only be re-enabled after the installer pipeline produces a complete non-stub executable.
